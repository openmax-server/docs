# Описание основ Websocket

Шлюз вебсокета является 1 из транспортов протокола MAX (всего их 2) и используется в данный момент только веб версией клиента (web.max.ru). Официальный сервер располагается по адресу `wss://ws-api.oneme.ru/websocket`, и его возможности крайне ограничены.

# Структура пакетов

Вот пример типичного пакета, который может поступить в вебсокет:

```json
{
    "ver": 11,
    "cmd": 0,
    "seq": 1,
    "opcode": 1,
    "payload": {
        "interactive": true
    }
}
```

- ver - версия протокола
- cmd - тип команды
- seq - порядковый номер пакета
- opcode - номер команды
- payload - данные пакета

## Подробнее про cmd

cmd это тип команды, от 0 до 4:

0 - всегда ставит клиент, и иногда сервер в некоторых командах
1 - успех выполнения операции (отправляет всегда сервер)
2 - ??? (скорее всего используется, но никогда не был замечен)
3 - ошибка при выполнении команды (отправляет всегда сервер)
